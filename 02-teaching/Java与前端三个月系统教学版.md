# Java 与前端三个月系统教学版

> 目标：把你从“知道一些零散概念”带到“具备可持续实战能力”。  
> 这份文档不是计划表，而是系统教学版知识文档。重点讲：前端、Java 后端、数据库、工程化，按 3 个月这个节奏应该真正掌握哪些内容、为什么学、学到什么程度、如何连接成完整项目能力。

---

# 一、三个月系统版和一周速成版的区别

## 一周速成版解决什么
- 快速建立全局地图
- 知道一个系统是怎么跑起来的
- 知道最重要的概念
- 能听懂大多数术语

## 三个月系统版解决什么
- 把知识从“知道”变成“会用”
- 把零散概念变成体系
- 把 demo 能力变成项目能力
- 把项目能力变成可面试、可转岗能力

所以这份文档更像一本“压缩教材”。

---

# 二、第一个月：前端基础 + Java 基础回捞

这个月目标只有一句话：

**把页面和接口这两条最基础的腿先站稳。**

---

## 2.1 前端基础系统学习

### 第一部分：HTML

HTML 的重点不是死记标签，而是理解“页面结构”。

你要会的核心：
- 容器：`div`、`span`
- 文本：`p`、`h1~h6`
- 表单：`input`、`textarea`、`button`、`form`
- 列表：`ul`、`li`
- 图片：`img`

### 你最终要达到的能力
- 能自己写登录页结构
- 能写列表页结构
- 能写聊天页结构

---

### 第二部分：CSS

你不需要一开始钻研特别复杂的 CSS，但这些必须会：

#### 1. 盒模型
- content
- padding
- border
- margin

#### 2. 布局
重点掌握：
- flex
- position
- width / height
- overflow

#### 3. 常见样式
- color
- background
- border-radius
- box-shadow
- font-size

### 为什么 CSS 对你重要
因为你从 iOS 来，很容易把 Web 页面想成 AutoLayout。这个类比没错，但 Web 的主战场是 Flex + 盒模型。

### 你要达到的能力
- 能自己做左右布局
- 能做顶部 / 底部固定结构
- 能做消息气泡和输入区样式

---

### 第三部分：JavaScript

这是前端逻辑的根。

你必须真正掌握：

#### 1. 基础语法
- 变量
- 条件判断
- 循环
- 函数

#### 2. 数据结构
- 对象
- 数组

#### 3. 常用数组方法
- map
- filter
- find
- reduce

#### 4. 异步编程
- Promise
- async / await
- try / catch

#### 5. 模块化
- import / export

### 为什么 JS 是关键
因为 React 最终也是 JS 在组织页面逻辑。

### 你要达到的能力
- 能自己写页面交互逻辑
- 能调接口并处理返回数据
- 能写清晰的数据处理代码

---

## 2.2 TypeScript 系统学习

### 为什么必须学
因为现代前端项目基本都离不开 TS。

### 必学部分
#### 1. 基本类型
- string
- number
- boolean
- any
- unknown

#### 2. interface / type
- 定义对象结构
- 定义接口返回结构
- 定义枚举型联合类型

#### 3. 泛型
- `Promise<T>`
- `ApiResponse<T>`

#### 4. 可选属性
- `name?: string`

### 要达到的能力
- 能为前端业务对象写类型
- 能为接口响应写泛型包装
- 能在项目中减少低级类型错误

---

## 2.3 Java 基础回捞

这一部分你有旧基础，所以重点不是从头学，而是“恢复工程手感”。

### 1. 面向对象
你必须重新熟悉：
- 类
- 对象
- 封装
- 继承
- 多态

### 2. 集合
重点：
- List
- Set
- Map
- ArrayList
- HashMap

### 3. 泛型
理解为什么接口返回、集合、工具类都要带泛型。

### 4. 异常
- checked exception 和 runtime exception 基础认知
- 自定义业务异常

### 5. 注解
因为 Spring Boot 大量靠注解驱动。

### 6. Stream / Lambda
你不用一开始玩很花，但要能看懂和写基础用法。

### 要达到的能力
- 能读懂 Java Web 项目
- 能写简单 Service 逻辑
- 能组织 DTO / Entity / VO 对象

---

# 三、第二个月：React / Next.js + Spring Boot / MySQL

这个月目标：

**把“页面”和“接口”真正打通，形成一个可运行的 Web 应用。**

---

## 3.1 React 系统学习

### 1. 组件思维
React 最大的价值，不是语法，而是组件化。

你要学会思考：
- 哪些是页面组件
- 哪些是业务组件
- 哪些是通用组件

### 2. props / state
这是 React 的根。

#### props
外部输入。

#### state
内部变化状态。

### 3. 常用 Hook
#### useState
管理局部状态。

#### useEffect
处理副作用，如请求接口。

### 4. 列表与条件渲染
这是几乎所有真实页面都会用到的。

### 5. 状态提升
当多个子组件要共享状态时，状态要往上提。

### 6. 表单处理
登录页、输入框、搜索框都离不开。

### 7. 组件拆分能力
你最终要做到：
- 聊天页能拆成 5~8 个组件
- 数据流向清晰
- 不把所有逻辑堆一个页面里

---

## 3.2 Next.js 系统学习

### 为什么要用
因为它是现代 React 产品开发很常见的工程框架。

### 重点掌握
#### 1. App Router
理解目录映射路由。

#### 2. page / layout
页面和布局分离。

#### 3. 客户端组件
你当前阶段先重点掌握交互组件即可。

#### 4. 环境变量
如何配置 API 地址。

#### 5. 项目结构
你要建立项目组织意识，而不是只会写一个 page 文件。

### 要达到的能力
- 能独立起一个 Next.js 项目
- 能组织登录页、聊天页、知识库页
- 能封装请求层和类型层

---

## 3.3 Spring Boot 系统学习

### 1. 基本概念
你要理解：
- IOC / DI 的目的是什么
- 为什么对象不用自己 new

### 2. Web 接口开发
#### 常用注解
- `@RestController`
- `@RequestMapping`
- `@GetMapping`
- `@PostMapping`
- `@RequestBody`
- `@PathVariable`

#### 接口设计
你要学会：
- 路径怎么设计
- 参数怎么设计
- 响应怎么统一

### 3. 三层架构
- Controller
- Service
- Mapper

### 4. 全局异常处理
为什么后端要统一返回错误，而不是直接崩栈。

### 5. 日志
要养成记录关键流程的习惯。

### 要达到的能力
- 能独立开发用户、会话、消息这几类接口
- 能写清晰的 Service 业务逻辑
- 能做统一返回和统一异常处理

---

## 3.4 MySQL 系统学习

### 1. 表设计能力
你需要学会从业务反推表。

例如聊天系统：
- user
- conversation
- message

### 2. SQL
必须会：
- select
- insert
- update
- delete
- where
- order by
- limit

### 3. 多表关系
理解：
- 一对多
- 外键逻辑关系（即使不一定真建外键）

### 4. 索引基础
知道为什么按 user_id / conversation_id 查时可能要加索引。

### 5. 事务基础
理解“要么都成功，要么都失败”。

### 要达到的能力
- 能设计支撑业务的表结构
- 能写核心查询 SQL
- 能分析简单性能问题

---

## 3.5 MyBatis / MyBatis-Plus

### 为什么学它
因为 Java 实战里很常用，而且适合快速业务开发。

### 要掌握
- Entity
- Mapper
- CRUD
- 条件查询
- 分页

### 要达到的能力
- 能把数据库数据映射到 Java 对象
- 能完成大部分业务 CRUD

---

# 四、第三个月：鉴权、Redis、工程化、以及向 Agent 过渡

这个月目标：

**把“能跑 demo”提升成“像一个真实项目”。**

---

## 4.1 JWT 鉴权系统

### 你必须吃透的流程
1. 登录接口校验账号密码
2. 生成 JWT
3. 前端保存 token
4. 请求头带 token
5. 后端拦截并解析 token
6. 获取当前用户信息
7. 接口按用户做数据隔离

### 为什么这部分很重要
因为一旦涉及多用户系统，它就是基础设施。

### 要达到的能力
- 能完整实现登录鉴权
- 能保证用户只访问自己的数据

---

## 4.2 Redis

### 什么时候用 Redis
- 登录态辅助
- 会话缓存
- 热点数据缓存
- 限流
- 任务状态

### 重点掌握
- string
- key 过期时间
- set / get
- 缓存设计基础

### 要达到的能力
- 知道什么时候该用 Redis
- 能写简单缓存逻辑

---

## 4.3 文件上传

### 为什么重要
Agent / RAG 一定会接文档。

### 你要学会
- 前端上传文件
- 后端接收 MultipartFile
- 保存文件信息
- 后续处理文档内容

---

## 4.4 Docker 与部署

### Docker 的核心价值
环境一致、部署稳定。

### 你要掌握
- Dockerfile
- 镜像和容器
- 端口映射
- 环境变量

### Docker Compose
把前端、后端、MySQL、Redis 一起编排起来。

### Nginx
统一入口和反向代理。

### 要达到的能力
- 能把自己的项目部署到服务器
- 能看日志和定位基础问题

---

## 4.5 工程化思维

### Git
- 提交习惯
- 分支基础
- commit message 规范化

### 日志
- 关键流程必须有日志
- 错误必须可追踪

### 配置管理
- 开发、测试、生产环境分离

### 要达到的能力
- 项目不只是“能写”，还“能维护”

---

# 五、怎么把这些知识串成真正的项目能力

这是最关键的一节。

你学这些知识，不是为了背八股，而是为了做出一个真实系统。

假设做一个 Agent Chat 产品：

## 前端部分
你要用到：
- HTML / CSS 做页面结构和布局
- React 组件化搭聊天界面
- TypeScript 定义消息 / 会话类型
- Next.js 组织整个项目

## 后端部分
你要用到：
- Java 实体类和 DTO / VO
- Spring Boot 提供接口
- MyBatis-Plus 查数据库
- JWT 完成登录鉴权
- Redis 提升性能和存状态

## 数据层
你要用到：
- user 表
- conversation 表
- message 表

## 工程化
你要用到：
- Git 做版本管理
- Docker 做部署
- Nginx 做网关入口

如果你能把上面这条链路打通，说明你已经不是“学了很多概念”，而是真正进入工程状态了。

# 六、你在三个月里真正该追求的“掌握标准”

## 前端掌握标准
- 能独立做登录页、聊天页、文件上传页
- 能封装组件
- 能调接口并处理状态

## Java 后端掌握标准
- 能独立写接口
- 能做 CRUD
- 能接数据库
- 能做登录鉴权
- 能处理异常和日志

## 工程化掌握标准
- 能把项目跑起来
- 能部署
- 能排查基础问题

## 转岗掌握标准
- 能做一个完整作品
- 能把项目讲清楚
- 能清楚解释技术选型和模块设计

# 七、我对你这个背景的最直接建议

你不是纯零基础，你最大的优势在于：
- 已经是工程师
- 已经会写代码
- 已经理解产品、交互、状态、网络这些概念

所以你最不该做的是：

- 把自己当纯小白，从头学很久
- 在基础概念上耗太久
- 看很多课，不做项目

你最该做的是：

- 前端学到“能做页面”
- Java 学到“能写服务”
- 立刻把它们放进一个 Agent 项目里

---

# 八、最后的结论

一周速成版，是帮你“通气血”；
三个月系统版，是帮你“长肌肉”。

真正有效的路径是：

1. 先用一周速成版建立完整认知
2. 再用三个月系统版把每块能力练扎实
3. 同时做一个 Agent 全栈项目，把知识变成作品

这样你转型的速度会远高于“先学半年理论再动手”的方式。
