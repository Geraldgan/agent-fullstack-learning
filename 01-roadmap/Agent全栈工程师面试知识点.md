# Agent 全栈工程师面试知识点

> 目标：帮助你从 iOS 开发转型时，知道面试官可能会问什么、你应该准备什么、如何把自己的项目讲清楚。

---

# 一、面试准备的总体思路

Agent 全栈岗位一般会从 4 个方向考你：

1. 前端基础与 React / Next.js
2. Java 后端与 Spring Boot
3. AI / Agent / RAG / Tool Calling
4. 项目实战与工程化能力

你不需要把每一块都答成专家水平，但一定要做到：
- 概念清楚
- 能结合项目说
- 不停留在背定义

---

# 二、前端面试高频知识点

## 1. React 基础
常问：
- React 组件是什么？
- props 和 state 的区别？
- useState 和 useEffect 怎么用？
- 父子组件怎么通信？
- 为什么列表渲染要加 key？

## 2. React 页面设计
常问：
- 你怎么拆分组件？
- 状态放哪里？
- 如何避免组件过于臃肿？
- 聊天页面的消息列表怎么设计？

## 3. Next.js
常问：
- 为什么用 Next.js？
- 它和纯 React 有什么区别？
- SSR / CSR 有什么区别？
- App Router 了解吗？

## 4. TypeScript
常问：
- interface 和 type 的区别？
- 泛型有什么用？
- 为什么前端项目要用 TS？

## 5. Web 基础
常问：
- HTTP 常见状态码有哪些？
- GET 和 POST 有什么区别？
- 什么是跨域？怎么解决？
- Cookie、LocalStorage、SessionStorage 区别？

# 五、项目面试最重要的问题

## 1. 你做的项目是什么？
回答结构建议：
1. 先讲项目定位
2. 再讲技术栈
3. 再讲你负责的模块
4. 再讲技术难点
5. 最后讲结果

## 2. 你在项目里的技术难点是什么？
可以从这些角度讲：
- 聊天上下文如何管理
- Tool Calling 如何设计执行器
- RAG 检索结果如何拼接
- 前后端如何联调
- 部署时遇到什么坑

## 3. 你最有价值的设计是什么？
比如：
- 统一消息结构设计
- 工具 schema + 执行器机制
- 用户与会话隔离
- 文档切片与引用展示

# 六、你应该准备的项目表达模板

## 项目介绍模板
我做了一个基于 Next.js + Spring Boot + MySQL + Redis 的 Agent 全栈平台，支持用户登录、会话管理、AI 对话、工具调用和知识库问答。我负责了前端页面开发、后端接口实现、数据库设计、大模型接入、Tool Calling 流程以及 Docker 部署。

## 技术亮点模板
- 前端使用 React / Next.js 实现聊天 UI、会话列表、文档上传等功能
- 后端采用 Spring Boot 分层架构，完成鉴权、会话管理、消息持久化
- 接入大模型 API 实现多轮对话，并支持工具调用
- 实现基础 RAG 流程，支持文档切片、检索和来源引用
- 使用 Docker 和 Nginx 完成部署

# 七、面试官最关注的其实是什么

不是你把概念背得多流利，而是：
- 你是不是真的做过
- 遇到问题时你怎么分析
- 你有没有完整的工程视角
- 你是不是只会调 API，还是会做完整产品

所以准备重点一定是：
- 把项目讲熟
- 把关键模块讲熟
- 把技术取舍讲清楚
