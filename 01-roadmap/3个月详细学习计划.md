# 3个月详细学习计划（iOS 开发转 Agent 全栈工程师）

> 适合对象：会 Objective-C / Swift，有 Java 基础，但缺少前端、后端实战和 Agent 工程经验。  
> 学习目标：12 周内完成前端 + 后端 + Agent 的最小闭环，做出 2~3 个可展示项目。

---

# 一、总目标拆解

12 周结束时，你应该达到以下状态：

## 能力目标
- 能独立写一个 Next.js 前端项目
- 能独立写一个 Spring Boot 后端项目
- 能完成前后端联调
- 能接入大模型 API
- 能实现简单 Tool Calling
- 能实现基础 RAG
- 能把项目部署到服务器

## 作品目标
至少产出：
1. AI Chat Web 前端
2. Spring Boot 聊天后端
3. 一个带工具调用和知识库的 Agent Demo

## 简历目标
能包装成：
- AI Chat Platform
- RAG Knowledge Assistant
- Agent Workflow Assistant

---

# 二、学习原则

## 原则 1：按项目倒推
每周都必须有产出，不要只看课。

## 原则 2：最小可运行优先
每学一个知识点，先做最小 demo。

## 原则 3：不要追求“学完再做”
前端、后端、Agent 三条线同时推进，但每周有主线。

## 原则 4：先会用，再补原理
你现在最缺的是工程落地感，不是理论深度。

---

# 三、时间安排建议

如果你是边工作边学，建议：

## 工作日
- 每天 2 小时
  - 1 小时学知识点
  - 1 小时写 demo / 做项目

## 周末
- 每天 4~6 小时
  - 2 小时系统学习
  - 2~4 小时实战编码

## 每周总投入
- 最低：14 小时
- 理想：18~22 小时

# 九、容易卡住的点与建议

## 1. React 很容易学得零散
建议：围绕聊天页面去学，不要单独刷太多题。

## 2. Spring Boot 很容易停留在 Demo 层
建议：尽快接数据库和鉴权，形成真实项目感。

## 3. Agent 很容易只会调 API
建议：一定要做 Tool Calling 和 RAG，不然竞争力不够。

## 4. 部署很容易拖到最后不做
建议：第 10 周开始就提前学 Docker，不要等全部做完再学。

# 十一、建议的学习节奏总结

你的目标不是成为“八股背诵型 Java 工程师”或者“纯前端切图工程师”，而是成为：

**能把 AI/Agent 产品真正从想法做到上线的工程师。**

所以最重要的节奏是：
- 每周有输出
- 每月有可展示成果
- 3 个月后有完整作品

先做出东西，再反补系统性。这个路径最适合你。
