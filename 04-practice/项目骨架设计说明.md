# 项目骨架设计说明

> 目标：告诉你接下来如果开始做第一个 Agent 全栈项目，前端和后端目录该怎么搭、模块该怎么拆。

---

# 一、推荐项目目录

```text
agent-chat-platform/
  frontend/
  backend/
  docs/
```

---

# 二、前端目录建议

```text
frontend/
  src/
    app/
      login/
        page.tsx
      chat/
        page.tsx
    components/
      Sidebar.tsx
      ChatInput.tsx
      MessageList.tsx
    services/
      request.ts
      chat.ts
    types/
      message.ts
      conversation.ts
    hooks/
    utils/
```

### 模块职责
- `app/`：页面
- `components/`：组件
- `services/`：接口请求
- `types/`：类型定义
- `hooks/`：复用逻辑
- `utils/`：工具函数

---

# 三、后端目录建议

```text
backend/
  src/main/java/com/example/agentplatform/
    controller/
    service/
    service/impl/
    mapper/
    entity/
    dto/
    vo/
    config/
    common/
    util/
```

### 模块职责
- `controller/`：接口层
- `service/`：业务层
- `mapper/`：数据库层
- `entity/`：数据库实体
- `dto/`：请求对象
- `vo/`：返回对象
- `config/`：配置
- `common/`：通用结构，如 Result
- `util/`：工具类

---

# 四、第一阶段先做哪些功能

## 前端
1. 登录页
2. 聊天页壳子
3. 消息列表
4. 输入框发送

## 后端
1. 登录接口
2. 会话列表接口
3. 消息列表接口
4. 发送消息接口

---

# 五、为什么这样拆

因为这样结构最清楚，也最适合你边学边做：
- 不会一开始就太复杂
- 又足够像真实项目
- 后面接 Tool Calling、RAG 时也方便扩展
